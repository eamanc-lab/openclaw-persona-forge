# user.md 模板

生成 user.md 时使用此模板。只填写已采集到的字段，未采集的整段省略（不要留空占位符）。

## 模板

```markdown
# User

## 基本信息

- **称呼**：[称呼]
- **角色**：[职业/身份]
- **行业**：[所在行业]
- **经验**：[工作年限或水平描述]

## 技术画像

- **技术栈**：[语言、框架、工具，逗号分隔]
- **使用场景**：[主要用 Agent 做什么]

## 偏好

- **沟通风格**：[简洁/详细、严肃/轻松、主动/被动]
- **语言**：[中文/英文/双语]

## 目标

[近期目标或长期方向，1-2 句话]

## 兴趣与关注

[技术以外的兴趣领域，或特别关注的方向]
```

## 示例

```markdown
# User

## 基本信息

- **称呼**：老王
- **角色**：全栈工程师
- **行业**：SaaS / B2B
- **经验**：8 年，前 5 年后端（Java），近 3 年全栈（React + Node）

## 技术画像

- **技术栈**：Java, TypeScript, React, Node.js, PostgreSQL, Docker
- **使用场景**：日常开发提效、代码审查、技术方案设计、写技术文档

## 偏好

- **沟通风格**：简洁直接，不需要过多解释，给方案不给选择题
- **语言**：中文

## 目标

带团队从单体架构迁移到微服务，同时保证业务不停。

## 兴趣与关注

关注 AI 辅助编程、DevOps 自动化、开源社区动态。
```

## 注意事项

- 文件名固定为 `user.md`，放在用户指定的目录
- 内容用 Markdown 格式，便于人类和 AI 阅读
- 不包含敏感信息（密码、API Key、身份证号等）
- 如果用户提供了敏感信息，**拒绝写入**并提醒
