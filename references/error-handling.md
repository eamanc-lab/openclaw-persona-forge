# 错误处理与降级策略

## 设计理念

> 任何错误都不应中断用户的创造流程。降级，不中断。

错误处理遵循三个原则：
1. **分类识别**：按错误来源分类，每类有独立的降级策略
2. **柔性降级**：功能不可用时自动切换到替代方案，不中断主流程
3. **透明沟通**：告诉用户发生了什么、影响了什么、替代方案是什么

## 错误分类与降级矩阵

### 类型 A：环境缺失

运行环境缺少必要的依赖。

| 错误场景 | 检测方式 | 降级策略 | 告知用户 |
|----------|---------|---------|---------|
| Python 3 不可用 | `python3 --version` 失败 | 跳过 gacha.py，改为从 10 个预设方向中随机选择 | "抽卡引擎需要 Python 3，已改用内置随机选择" |
| Node.js / bun 不可用 | `npx -y bun --version` 失败 | 跳过自动生图，只输出提示词文本 | "生图引擎需要 Node.js + bun，已输出提示词供手动使用" |

**降级后的替代行为**：

```markdown
## 抽卡降级（无 Python）
从 SKILL.md 的 10 个预设方向中，由 Claude 随机选择 1 个作为基础方向，
同时随机组合气质关键词和道具。结果不如 gacha.py 的 168 万种组合丰富，
但足以启动锻造流程。

## 生图降级（无 Node.js / bun）
输出完整的提示词文本，格式如下：

**头像提示词（可复制到以下平台手动生成）**：
- Google Gemini：直接粘贴
- ChatGPT（DALL-E）：直接粘贴
- Midjourney：粘贴后加 `--ar 1:1 --style raw`

> [完整英文提示词]
```

### 类型 B：API / 网络错误

外部服务调用失败。

| 错误场景 | 检测方式 | 降级策略 | 告知用户 |
|----------|---------|---------|---------|
| API Key 未配置 | 脚本输出 "No API key found" | 跳过生图，输出提示词文本 | "未检测到 API Key，已输出提示词。配置方法见下方" |
| API 调用失败（网络/限流/余额） | 脚本返回非零退出码 | 重试 1 次，仍失败则输出提示词文本 | "生图服务暂时不可用，已输出提示词供手动使用" |
| API Key 无效/过期 | 脚本输出 "401" 或 "invalid key" | 跳过生图，输出提示词文本 + 配置指引 | "API Key 无效，请检查配置" |

**API Key 缺失时的配置指引**（随错误一起输出）：

```markdown
**快速配置 API Key**（三选一）：

| Provider | 命令 |
|----------|------|
| Google Gemini（推荐） | `mkdir -p ~/.baoyu-skills && echo 'GOOGLE_API_KEY=你的Key' > ~/.baoyu-skills/.env` |
| OpenAI | `mkdir -p ~/.baoyu-skills && echo 'OPENAI_API_KEY=你的Key' > ~/.baoyu-skills/.env` |
| 阿里云 DashScope | `mkdir -p ~/.baoyu-skills && echo 'DASHSCOPE_API_KEY=你的Key' > ~/.baoyu-skills/.env` |

获取 Key：[Google AI Studio](https://aistudio.google.com/apikey) / [OpenAI](https://platform.openai.com/api-keys) / [DashScope](https://dashscope.console.aliyun.com/apiKey)
```

### 类型 C：运行时异常

脚本本身出错。

| 错误场景 | 降级策略 | 告知用户 |
|----------|---------|---------|
| gacha.py 输出格式异常 | 解析失败时，从 10 个预设方向中随机选择 | "抽卡结果解析失败，已改用内置随机" |
| 生成的图片文件不存在 | 输出提示词文本 | "图片生成异常，已输出提示词供手动使用" |
| 任何未预期错误 | 记录错误信息，继续主流程 | "遇到了一个问题：[错误简述]。已跳过该步骤继续" |

## 执行规范

### 每个可能出错的步骤，按此模式执行：

```
1. 尝试执行
2. 检查结果
3. 如果失败：
   a. 识别错误类型（A/B/C）
   b. 执行对应的降级策略
   c. 向用户输出：[发生了什么] + [影响范围] + [替代方案]
   d. 继续主流程，不中断
```

### 错误信息输出格式

当任何步骤降级时，用以下格式告知用户：

```markdown
> ⚠️ **[步骤名] 已降级**
> 原因：[一句话说明]
> 影响：[哪个功能受限]
> 替代：[正在使用什么替代方案]
> 修复：[可选，怎么恢复完整功能]
```

示例：

```markdown
> ⚠️ **头像生成已降级**
> 原因：未检测到 GOOGLE_API_KEY
> 影响：无法自动生成头像图片
> 替代：已输出完整提示词，可复制到 Gemini 手动生成
> 修复：运行 `mkdir -p ~/.baoyu-skills && echo 'GOOGLE_API_KEY=你的Key' > ~/.baoyu-skills/.env`
```

## 关键原则

1. **永远不要因为辅助功能失败而中断主流程**——人设方案（文本）是核心价值，头像是锦上添花
2. **降级信息要可操作**——不只是说"出错了"，要说"怎么修"
3. **一次降级不影响后续步骤**——Step 5 降级了，Step 6 照常输出（头像标注为"待生成"）
